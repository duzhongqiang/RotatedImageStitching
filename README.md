## 1. 整体流程

程序采用**增量式**拼接策略：以第一帧为坐标原点，每当摄像头旋转到位后采集一帧关键帧，逐步将其融合进全景图，而不是等所有帧采集完再批量处理。

```
摄像头帧
   │
   ▼
[方向预处理]  ← 左旋时水平翻转
   │
   ▼
[关键帧检测]  ← 位移够大 且 方向正确 才触发
   │
   ▼
[柱面投影]    ← 消除平面透视畸变
   │
   ▼
[SIFT 特征提取 & FLANN 匹配]
   │
   ▼
[RANSAC 单应矩阵估计]
   │
   ▼
[累积变换 → 全景坐标系]
   │
   ▼
[动态扩展画布 + 羽化融合]
   │
   ▼
全景图（实时更新）
```

## 2. 柱面投影

### 问题背景

摄像头绕竖直轴旋转时，相邻帧之间存在透视变形：离光轴越远的区域，图像内容被压缩得越厉害。若直接用平面单应矩阵拼接，全景图边缘会出现明显弯曲。

### 投影公式

将平面图像坐标 `(xd, yd)` 映射到柱面图像坐标 `(xs, ys)`：

```
xs = f · tan((xd - cx) / f) + cx
ys = (yd - cy) · √(((xd - cx) / f)² + 1) + cy
```

其中 `f` 为焦距（像素），`(cx, cy)` 为图像中心。

### 直觉理解

柱面投影把"平面相机看到的场景"重新展开到一个虚拟圆柱面上。展开后，纯水平旋转引起的位移变成近似**纯平移**，从而让后续的单应矩阵估计更准确、全景图更平直。

```
平面图像（有透视弯曲）        柱面展开图（近似平移）
  ┌──────────────────┐         ┌──────────────────┐
  │  /          \    │   →→→   │                  │
  │ /            \   │         │   平直的场景内容   │
  │/              \  │         │                  │
  └──────────────────┘         └──────────────────┘
```

焦距 `f` 的选取：
- 若已知真实焦距（毫米），需换算为像素：`f_px = f_mm / sensor_width_mm × image_width_px`
- 未提供时默认用帧宽的 0.7 倍估算（`f = w × 0.7`）

## 3. 特征检测与匹配

### SIFT 特征检测

使用 `cv2.SIFT_create()` 在柱面图像上提取关键点和 128 维描述符。SIFT 具有尺度不变性和旋转不变性，适合在不同曝光、轻微抖动的相邻帧间找到稳定的对应点。

### FLANN 近似最近邻匹配

使用 `FlannBasedMatcher`（KD-Tree，5 棵树，50 次检查）对描述符做 K=2 的近似最近邻搜索，然后用 **Lowe 比率测试** 过滤歧义匹配：

```python
good_matches = [m for m, n in matches if m.distance < 0.75 * n.distance]
```

只有当最近邻距离明显小于次近邻时（比率 < 0.75），才认为该匹配可靠。

## 4. 关键帧自动检测

每 3 帧检测一次（降低 CPU 占用），判断当前帧是否值得加入全景。检测逻辑：

```
1. 提取当前帧特征，与上一关键帧匹配
2. 若匹配点 < min_matches → 跳过（重叠不够）
3. 用匹配点估计单应矩阵 H
4. 计算图像中心经 H 变换后的水平位移 signed_dx：
       signed_dx = H([cx, cy])[0] - cx
5. signed_dx ≤ 0 → 摄像头向反方向移动，丢弃
6. signed_dx < min_shift_px → 位移太小，还没转到位
7. signed_dx > max_shift_px → 转太快，重叠不足，丢弃
8. 通过以上所有检查 → 采集为关键帧
```

`signed_dx` 的符号：
- **正值**：中心点向右偏移，即摄像头向右旋转
- **负值**：中心点向左偏移，即摄像头向左旋转（左旋模式下帧已翻转，因此仍为正值）

## 5. 单应矩阵与累积变换

### 逐帧单应矩阵

对每对相邻关键帧（上一帧 F_{n-1}，当前帧 F_n），用 RANSAC 求解单应矩阵 `H_to_prev`，使得：

```
pts_in_F(n-1) ≈ H_to_prev · pts_in_F(n)
```

即 `H_to_prev` 将当前帧坐标系映射到上一帧坐标系。RANSAC 在迭代中随机采样 4 对点，找到内点（重投影误差 < 4px）最多的解，有效剔除误匹配。

### 累积变换

为了将第 n 帧直接定位到全景坐标系（第 1 帧坐标系），利用链式法则累积各帧变换：

```
H_abs(1) = I                        （第 1 帧是坐标原点）
H_abs(2) = H_abs(1) · H_to_prev(2)
H_abs(3) = H_abs(2) · H_to_prev(3)
...
H_abs(n) = H_abs(n-1) · H_to_prev(n)
```

每次只需做一次矩阵乘法，避免多次透视变换带来的误差叠加。

## 6. 动态画布扩展

### 计算新帧落点

将新帧的四个角点通过累积变换投影到全景坐标系，与现有画布边界取并集，确定需要多大的新画布：

```python
corners = [[0,0], [w,0], [w,h], [0,h]]
corners_in_panorama = perspectiveTransform(corners, H_final)

x_min = min(corners_in_panorama[:, 0].min(), 0)
x_max = max(corners_in_panorama[:, 0].max(), panorama_width)
```

### 两种扩展路径

| 情形 | 触发条件 | 操作 | 性能 |
|------|---------|------|------|
| 向右扩展 | `x_max > 当前宽` | numpy 零填充 | O(新增面积) |
| 向左扩展 | `x_min < 0` | 对整张全景做平移 warpPerspective | O(全景面积) |

`_canvas_offset` 记录当前画布原点相对于第 1 帧坐标系的偏移量，每次向左扩展时累加，保证后续帧的 `H_final` 始终正确落在画布范围内。

> **为何向右拼接更高效**：向左扩展时需要对已有全景整体做透视变换，全景越大代价越高。向右扩展只需追加零数组，已有像素从不移动。

## 7. 羽化融合

### 权重图生成

对每帧做**距离变换**生成权重图：每个像素的权重 = 该像素到最近黑色边界的欧氏距离。离中心越近权重越高，边缘权重趋向 0，形成自然过渡。

```
像素权重 = distanceTransform(非黑区域掩膜, DIST_L2)

示意（截面）：
  0  0  0  1  2  3  3  3  2  1  0  0  0   ← 权重
  ▓  ▓  ▓  ░  ░  ░  ░  ░  ░  ░  ▓  ▓  ▓   ← 图像边缘
```

### 加权融合公式

在重叠区域，按各自权重做加权平均，避免硬边缝合线：

```
pixel_out = (pixel_panorama × w_panorama + pixel_new × w_new)
            / (w_panorama + w_new + ε)
```

`ε = 1e-6` 防止除零。融合后用 `max` 更新权重图，保留覆盖范围最广的权重信息。

## 8. 旋转方向处理

### 核心思路：翻转等价

左旋和右旋的全景拼接在数学上等价，区别仅在于场景在画面中的移动方向。

**向左旋转** 时，每帧在读入后立即水平翻转：

```python
frame = cv2.flip(frame, 1)   # 水平翻转
```

翻转后，"向左旋转"在翻转图像中的表现与"向右旋转"在原图中完全一致，整个拼接流程无需任何修改。保存前再翻转一次还原真实方向：

```python
result = cv2.flip(result, 1)   # 还原
```

### 方向检测过滤

`_auto_detect` 中检查 `signed_dx` 的符号（翻转后始终期望正值），若摄像头朝反方向抖动则丢弃该帧，避免误触发：

```python
if signed_dx <= 0:
    return   # 反向运动，丢弃
```

## 9. 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--focal` / `-f` | 自动估算 | 焦距（像素）。已知真实焦距时手动指定，可提升柱面投影精度 |
| `--min-shift` | `0.08` | 触发关键帧的最小位移（帧宽比例）。值越小采帧越频繁，拼缝越少但速度更慢 |
| `--direction` / `-d` | `right` | 旋转方向：`right`（向右）或 `left`（向左）|
| `--camera` / `-c` | `0` | 摄像头设备 ID |
| `--output` / `-o` | `realtime_panorama.jpg` | 输出文件路径 |

内部阈值（构造函数参数）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_shift_ratio` | `0.60` | 最大允许位移（帧宽比例），超过说明旋转过快、重叠不足 |
| `min_matches` | `10` | 特征匹配点最低数量，低于此值放弃该帧 |
