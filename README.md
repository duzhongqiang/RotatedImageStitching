# 全景拼接工具

基于 OpenCV Stitcher 的序列图像全景拼接，支持任意旋转拍摄（偏航 / 俯仰 / 滚转）。

## 快速开始

```bash
# 旋转拍摄数据（默认，推荐）
python run_stitch.py --input ./images --output panorama.jpg

# 预过滤冗余帧，加快速度
python run_stitch.py --input ./images --output panorama.jpg --keyframe-shift 50

# 平移扫描场景（文档、平面物体）
python run_stitch.py --input ./images --output panorama.jpg --mode scans
```

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--input` / `-i` | 必填 | 输入图像文件夹路径（按文件名自然排序） |
| `--output` / `-o` | `panorama.jpg` | 输出全景图路径 |
| `--mode` | `panorama` | 拼接模式：`panorama`=旋转拍摄，`scans`=平移扫描 |
| `--keyframe-shift` | `0`（不筛选） | 关键帧阈值（像素）：相邻帧位移超过此值才纳入拼接 |

支持的图像格式：`.jpg` `.jpeg` `.png` `.bmp` `.tiff` `.webp`

## 算法流程

```
输入图像序列
      │
      ▼
[关键帧筛选]  ← 相位相关法过滤位移不足的冗余帧（可选）
      │
      ▼
[SIFT 特征检测]  ← 每张图提取关键点 + 128 维描述符
      │
      ▼
[FLANN 特征匹配]  ← K=2 近邻 + Lowe 比率测试过滤歧义匹配
      │
      ▼
[RANSAC 单应矩阵估计]  ← 每对相邻图像估计 3×3 单应矩阵 H
      │
      ▼
[Bundle Adjustment]  ← 全局优化所有相机旋转矩阵 R
      │
      ▼
[球面 / 仿射投影]  ← 将各图像重投影到统一坐标系
      │
      ▼
[增益补偿]  ← 均衡相邻图像曝光差异
      │
      ▼
[多频段融合]  ← 拉普拉斯金字塔消除拼缝
      │
      ▼
全景图
```

---

## 算法原理

### 1. 关键帧筛选（相位相关）

对相邻帧计算**相位相关**，在频域估计整体平移量 `(dx, dy)`：

```
F₁(u,v) · conj(F₂(u,v))
R(u,v) = ─────────────────────
         |F₁(u,v) · conj(F₂(u,v))|

(dx, dy) = argmax IFFT(R)
```

相位相关本质是归一化互功率谱的逆变换，对亮度变化和噪声有较强鲁棒性，且只需两次 FFT，计算量远低于 SIFT。只有当位移 `√(dx²+dy²) ≥ keyframe_shift` 时才保留该帧。

---

### 2. SIFT 特征检测

SIFT（Scale-Invariant Feature Transform）在图像的**尺度空间**（高斯差分金字塔 DoG）中寻找极值点作为关键点，并为每个关键点计算主方向和 128 维梯度直方图描述符。

```
DoG(x,y,σ) = G(x,y,kσ) * I − G(x,y,σ) * I
```

描述符对尺度、旋转、光照变化具有不变性，适合相邻帧间存在轻微抖动或曝光差异的场景。

---

### 3. FLANN 匹配与 Lowe 比率测试

用 FLANN（近似最近邻库，KD-Tree）对每个描述符搜索 K=2 个最近邻，再用 **Lowe 比率测试**过滤歧义匹配：

```
保留条件：d₁ / d₂ < ratio（默认 0.75）
```

当最近邻距离明显小于次近邻时，说明该匹配具有唯一性，可信度高。该测试可过滤掉约 90% 的错误匹配，同时保留 95% 以上的正确匹配。

---

### 4. RANSAC 单应矩阵估计

对每对图像的匹配点集，用 RANSAC 鲁棒估计 3×3 单应矩阵 `H`：

```
[x']     [h₀₀ h₀₁ h₀₂] [x]
[y']  ~  [h₁₀ h₁₁ h₁₂] [y]
[1 ]     [h₂₀ h₂₁  1 ] [1]
```

RANSAC 每轮随机采样 4 对点求解 `H`，统计所有匹配中重投影误差 `< 4px` 的内点数，取内点最多的解。对于存在移动前景或重复纹理的场景，RANSAC 能有效排除离群误匹配。

---

### 5. Bundle Adjustment

逐对单应矩阵存在误差累积问题：第 1→2 帧的误差会传递到 1→3、1→4……帧数越多，末端偏差越大。Bundle Adjustment 将所有相机参数纳入统一优化，最小化**全局重投影误差**：

```
        N   Mᵢ
E(R) = Σ   Σ  ‖xᵢⱼ − π(Rᵢ, Kᵢ, Xⱼ)‖²
       i=1 j=1
```

其中 `Rᵢ` 为第 i 帧旋转矩阵，`Kᵢ` 为相机内参，`Xⱼ` 为三维场景点，`π` 为投影函数。使用 Levenberg-Marquardt 算法迭代求解。Bundle Adjustment 后各帧姿态误差均匀分布，不再随帧数线性增长。

---

### 6. 球面投影

`panorama` 模式将图像从平面坐标重投影到球面坐标，消除旋转拍摄带来的透视形变。给定焦距 `f`，平面坐标 `(x, y)` 到球面坐标 `(θ, φ)` 的映射：

```
θ = arctan((x − cx) / f)
φ = arctan((y − cy) / f)
```

展开后，纯旋转引起的图像偏移在球面上变为近似**匀速平移**，使拼接后的全景图在大角度旋转下保持均匀比例，不出现边缘拉伸。

---

### 7. 增益补偿

不同帧可能因自动曝光或光照变化存在亮度差异，直接拼接会产生明显明暗分界线。增益补偿对每张图像估计一个标量增益 `gᵢ`，使重叠区域的亮度尽量一致：

```
最小化：Σ σᵢⱼ · (gᵢ Iᵢ − gⱼ Iⱼ)²
```

其中 `σᵢⱼ` 为两图重叠区域的面积权重，通过线性方程组求解各 `gᵢ`。

---

### 8. 多频段融合

增益补偿后仍可能有高频拼缝（颜色、纹理跳变）。多频段融合通过**拉普拉斯金字塔**在不同频率分别融合：

```
低频（大结构）：宽过渡带，平滑颜色过渡
高频（细节） ：窄过渡带，保留纹理锐度
```

具体步骤：
1. 对每张图像和其权重掩膜分别构建高斯金字塔
2. 从高斯金字塔计算拉普拉斯金字塔（各层细节）
3. 在每层按权重混合各图像的细节
4. 从底层向上重建最终图像

相比简单的羽化融合，多频段融合能同时避免颜色鬼影（低频平滑）和纹理模糊（高频保真）。

## 两种模式对比

| | `panorama` | `scans` |
|---|---|---|
| 适用场景 | 相机原地旋转（偏航/俯仰/滚转） | 相机平移扫描（文档、地图） |
| 投影方式 | 球面投影 | 仿射变换 |
| Bundle Adjustment | 优化旋转矩阵 R | 优化仿射参数 |
| 典型误差来源 | 焦距估计偏差 | 非平面场景的透视形变 |

## 关键帧筛选建议

| 场景 | 建议阈值 |
|------|---------|
| 帧间重叠 ~70%，帧数 <30 | `0`（不筛选，质量最优） |
| 帧间重叠 ~70%，帧数 >50 | `50`～`80` px |
| 帧间重叠 ~50%，帧数较多 | `30`～`50` px |

阈值过大会丢失必要重叠帧，导致 Stitcher 报"图像数量不足"错误；阈值过小则效果与不筛选相近。
